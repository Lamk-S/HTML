¿Qué es la carpeta raíz (root)?

La carpeta raíz es la carpeta principal de un proyecto web. Todo lo que ve el navegador empieza aquí.

Piénsala como la puerta de entrada de tu sitio.

Estructura que vamos a usar

En este curso básico trabajaremos solo con la siguiente estructura:

root/
├─ index.html
├─ assets/
│  ├─ css/
│  ├─ js/
│  └─ img/
└─ pages/

No es la única forma de organizar un proyecto, pero es una forma simple, clara y correcta para empezar.

1. index.html

Es el archivo más importante del proyecto.

  • Cuando escribes un dominio (ejemplo: www.miweb.com), el navegador busca automáticamente index.html.
  • Por eso siempre vive en la carpeta raíz.

¿Qué contiene?

  • La estructura principal del sitio.
  • Los enlaces a los archivos CSS y JavaScript.
  • El contenido inicial que ve el usuario.

2. assets/

La carpeta assets guarda los recursos que usa la página, pero que no son páginas HTML.

Sirve para mantener el proyecto ordenado y fácil de entender.

2.1 assets/css/

Aquí van los archivos CSS.

  • El CSS se encarga de los colores, tamaños y estilos.
  • Separar CSS de HTML hace el código más limpio.
<link rel="stylesheet" href="/assets/css/style.css">

2.2 assets/js/

Aquí van los archivos JavaScript.

  • JavaScript permite que la página tenga interacción.
  • Ejemplo: botones, validaciones, mensajes.
<script src="/assets/js/main.js"></script>

2.3 assets/img/

Aquí se guardan las imágenes del sitio.

  • Logos, fotos, íconos, banners.
  • Tenerlas juntas evita perder archivos.
<img src="/assets/img/logo.png" alt="Logo">

3. pages/

La carpeta pages contiene otras páginas HTML del sitio.

Ejemplos comunes:

  • about.html → información del sitio
  • contact.html → formulario de contacto

Esto evita llenar la carpeta raíz con muchos archivos.

Idea clave para recordar

  • index.html → página principal
  • assets → recursos (CSS, JS, imágenes)
  • pages → otras páginas HTML

Si entiendes esto, ya entiendes la base de cualquier proyecto web.
